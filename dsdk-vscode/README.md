# Code in Motion VS Code Extension

A VS Code extension that integrates with Code in Motion to provide seamless task execution and workspace management.

## Features

- **Automatic Task Discovery**: Automatically discovers and loads SDK tasks from `tasks.json`
- **Task View Sidebar**: Browse and run tasks from a dedicated sidebar view
- **Task Provider**: Registers custom task type `sdk` for VS Code task system
- **Quick Execution**: Run any SDK task from the Command Palette
- **Status Display**: Visual feedback on task execution in the Output panel
- **Workspace Detection**: Automatically activates when `.workspace` marker is found

## Installation

### From Source

```bash
cd dsdk-vscode
npm install
npm run esbuild
```

### Building VSIX Package

```bash
npm run vscode:prepublish
npx vsce package
```

## Usage

### Running Tasks

1. **From Sidebar**: Open the SDK Tasks view on the activity bar and click any task to run it
2. **From Command Palette**: 
   - `Ctrl+Shift+P` (or `Cmd+Shift+P` on macOS)
   - Search for "Run Task" or specific SDK task names
   - Select and execute

3. **From Task Execution**:
   - Use VS Code's built-in Tasks: Run Task (`Ctrl+Shift+B` for build tasks)
   - Tasks are categorized as "Build" or "Test" groups

### Commands

- **cim: Refresh Tasks** (`cim.refreshTasks`): Reload tasks from `tasks.json`
- **cim: Run Task** (`cim.runTask`): Quick picker to select and run a task
- **cim: Edit Task Configuration** (`cim.editTasks`): Open `tasks.json` in editor

## Configuration

### Tasks File Location

Tasks are automatically read from `.vscode/tasks.json` in the workspace root. This file is generated by the cim CLI tool when running:

```bash
cim makefile
```

### Task Definition Format

Tasks use the custom `sdk` task type:

```json
{
  "version": "2.0.0",
  "tasks": [
    {
      "type": "sdk",
      "label": "SDK: Build",
      "target": "sdk-build",
      "command": "make",
      "args": ["sdk-build"],
      "group": {
        "kind": "build",
        "isDefault": true
      }
    }
  ]
}
```

## Troubleshooting

### Tasks Not Showing

1. Ensure `.workspace` exists in workspace root
2. Run `cim makefile` to generate `tasks.json`
3. Reload VS Code window (`Ctrl+Shift+P` ‚Üí "Developer: Reload Window")

### Output Not Showing

- Check cim Output Channel: View ‚Üí Output ‚Üí "cim"
- Verify `tasks.json` exists and is valid JSON
